Introduction

  • Node.js
  • NPM node package manager
  • io.js create a fork of node.js project

  • Node.js 4.0 released, foundation of both node.js and io.js

  • Apache: every request is single-thread, thread waiting for file system/data base finish the last request before it can do anything(blocking)

  • Node.js: non-blocking event driven IO, only single thread && multitask, asynchroniously no need for waiting

How to haddle json file with node.js

Definition

  • JSON: JavaScript Object Notation
  • a lightweight data format which has become a web standard followed by a majority of web developers across the globe
  • represented as a hash of properties and values(for e.g. an Object) or as a list of values (for e.g. an Array).

Handling JSON Files in Node.js

Step1

  • Create a dummy JSON file: dummy.json, and save the same file as demo.txt.

Step2

  • synchronous: one-way execution wherein there is a single flow which executes the JSON file line by line. The file will move to the next statement only after the statement has completed.
  • asynchronous: the flow-control will move to the next line by displaying the concerns over the statement’s completion

Step3

  • synchronous:

    • convert JSON to a String

    • convert a String to a JSON object

    • trim(): when there are some chances of having any extra space within the JSON string

    • automatically reading a JSON file

    • Adding a new element to an existing JSON object

    • Traversing through each element within a JSON object

    • check whether the JSON has a specific key
